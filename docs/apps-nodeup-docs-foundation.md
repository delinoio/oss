# apps-nodeup-docs-foundation

## Scope
- Project/component: `nodeup` static documentation app foundation contract
- Canonical path: `apps/nodeup-docs`

## Runtime and Language
- Runtime: Rspress static documentation app
- Primary language: Markdown and TypeScript configuration
- Build toolchain: Rspress/Rsbuild-style tooling, aligned with `docs/repository-defaults.md`.
- Deployment target: Cloudflare Pages static-site deployment by default.

## Users and Operators
- External users reading `nodeup` installation and usage documentation
- Maintainers reviewing `nodeup` documentation changes
- Release operators publishing static documentation artifacts

## Interfaces and Contracts
- Stable package name: `nodeup-docs`
- Stable local dev port: `46250`
- Stable route IDs:
  - `/` -> overview landing page
  - `/guide/getting-started` -> initial setup guide
- Documentation content must stay aligned with the `nodeup` CLI, release, installer, and shim contracts in `docs/project-nodeup.md` and `docs/crates-nodeup-foundation.md`.
- User-facing guidance must not contradict release verification requirements for `SHA256SUMS`, Sigstore bundle sidecars, or supported host targets.
- Breaking route changes require updates to this contract, `docs/project-nodeup.md`, and any deployment or link references in the same change.

## Storage
- Source content is versioned in `apps/nodeup-docs`.
- Build artifacts are generated by Rspress under `apps/nodeup-docs/doc_build` and published as static assets through Cloudflare Pages.
- The app does not own persisted user data or upload/download object storage.

## Security
- Public content must not include secrets, internal credentials, or unpublished release material.
- Install and release documentation must preserve documented verification requirements.
- Cloudflare Pages publishing credentials must be managed by deployment infrastructure, not committed app files.

## Logging
- Local and CI builds should emit actionable Rspress build logs.
- Deployment logs should include build status and artifact publishing status without exposing secrets.

## Build and Test
- Local development: `pnpm --filter nodeup-docs dev`
- Production build: `pnpm --filter nodeup-docs build`
- Local validation: `pnpm --filter nodeup-docs test`
- CI alignment: `node-nodeup-docs-test`
- Cloudflare Pages output directory: `apps/nodeup-docs/doc_build`
- Root workspace preparation runs `prepare:app`; `nodeup-docs` must keep a non-interactive `prepare:app` script.

## Dependencies and Integrations
- Integrated into the root pnpm workspace through `apps/*`.
- Uses `@rspress/core` for static documentation generation.
- Depends on `docs/project-nodeup.md` and `docs/crates-nodeup-foundation.md` as source-of-truth contract references.
- Deployment contract follows the repository default Cloudflare Pages static-site target.

## Change Triggers
- Update `docs/project-nodeup.md` and this file when routes, runtime, publishing workflow, validation commands, or user-facing `nodeup` documentation contracts change.
- Update `apps/AGENTS.md` when app-domain validation or policy rules change.
- Update `docs/README.md` when adding, renaming, or removing app-domain contract documents.
